"Complex systems are systems whose behaviour is intrinsically difficult to model due to the dependencies, competitions, relationships, or other interactions between their parts or between a given system and its environment." Biology is packed with complex systems which are often challenging to comprehend. Genet represents genetic designs as networks. This representation allows analysis and transformations to be applied to tailor visualisation based on requirements.

Genet uses a neo4j server to store a graph containing multiple graphs where one of these may pertain to a design. Encoding designs as networks enable the data to be transformed into more comprehensible structures by different users. Tailored views allow connections to be made that may not be immediately apparent.

FEATURES

All sub-tools can be accessed from the toolbar above.
  • Graph Admin - The section for more administrative functions on the graph, such as adding and removing data.
  • Visualiser - A tool purely for a visualisation tool and allows for many automated and manual changes to the representation. Note this may be very slow with large graphs.
  • Editor - A tool which visualises designs but also allows manual editing of the design. Editing can be done on views which are propogated back into the original dataset.
  • Database - A tool which visualises the underlying network representation of several existing unified databases to present this information as a single entity.

USAGE

  • 1. Open Graph Admin
  • 2. In the "Add New Graph" section, upload, paste, or point to a design file.
  • 3. Pick a tool from the features. Each tool revolves around a visual representation of the data but with different mechanisms for presentation and analysis.

FUTURE

Some high-level descriptions of future work.
  • 1. The addition of higher-order processes for the enhancement and validation of genetic designs.
  • 2. The ability to generate designs from abstract descriptions.
